CREATE TABLE usuarios (
    id INT AUTO_INCREMENT PRIMARY KEY,
    email VARCHAR(100) UNIQUE NOT NULL,
    senha VARCHAR(255) NOT NULL,
    status_assinatura ENUM('ativo','inativo') DEFAULT 'ativo',
    data_expiracao DATE NOT NULL,
    criado_em TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);


🔒 Segurança:
- senha deve ser armazenada com hash seguro (bcrypt/argon2).
- dois_fatores define o método de validação (nenhum, e-mail, SMS ou app autenticador).
- token_2fa guarda o código temporário enviado ao usuário.
- token_expira garante que o código só seja válido por alguns minutos.

2. Tabela cliente (dados cadastrais)
CREATE TABLE clientes (
    id INT AUTO_INCREMENT PRIMARY KEY,
    usuario_id INT NOT NULL,              -- vínculo com a tabela usuario
    nome VARCHAR(100) NOT NULL,
    documento VARCHAR(20),                -- CPF ou RG
    endereco VARCHAR(200),
    telefone VARCHAR(20),
    FOREIGN KEY (usuario_id) REFERENCES usuarios(id)
);



Fluxo prático de login com 2FA
- Usuário entra com email + senha → sistema valida na tabela usuario.
- Se dois_fatores ≠ nenhum, o sistema gera um token_2fa e salva com token_expira.
- O token é enviado por email/SMS/app.
- Usuário digita o código → sistema valida se corresponde ao token_2fa e se está dentro do prazo.
- Se válido → acesso liberado.

👉 Assim você terá um sistema de login robusto, com senha criptografada e segunda camada de segurança.
